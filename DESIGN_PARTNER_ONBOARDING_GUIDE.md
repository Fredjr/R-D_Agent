## üéØ What is R&D Agent?

R&D Agent is an AI-powered research platform that helps you:
- **Organize literature**: Search PubMed, save papers, create collections
- **Track research questions**: Build a hierarchy of questions and sub-questions
- **Manage hypotheses**: Link hypotheses to questions with evidence
- **Collaborate**: Share projects with your lab or collaborators

**Our Mission**: Turn papers into a living research plan.

---

## ‚ú® What's New: Questions Tab

The **Questions Tab** is a new feature that helps you organize your research around questions and hypotheses. Think of it as your research roadmap.

### Key Features

#### 1. **Research Question Hierarchy**
- Create main research questions
- Add sub-questions to break down complex topics
- Organize questions in a tree structure
- Track status (Exploring, Investigating, Answered, Parked)
- Set priority levels (Low, Medium, High, Critical)

#### 2. **Hypothesis Tracking**
- Link hypotheses to specific questions
- Track hypothesis status (Proposed, Testing, Supported, Rejected)
- Set confidence levels
- View supporting vs contradicting evidence counts

#### 3. **Evidence Linking**
- Link papers from your project as evidence
- Choose evidence type:
  - **Questions**: Supports, Contradicts, Neutral, Context, Methodology
  - **Hypotheses**: Supports, Contradicts, Neutral
- Rate evidence strength (Weak, Moderate, Strong)
- Add key findings to each evidence link

#### 4. **Visual Organization**
- See your entire research plan at a glance
- Expand/collapse question branches
- Color-coded status badges
- Evidence count indicators

## üìñ Step-by-Step Tutorial

### Tutorial 1: Create Your First Research Question

**Scenario**: You're researching "How does exercise affect cognitive function in aging?"

**Steps**:
1. Click **"Add Main Question"** button
2. Enter your question: "How does exercise affect cognitive function in aging?"
3. Select question type: **Main**
4. Set status: **Exploring**
5. Set priority: **High**
6. Click **"Create Question"**

‚úÖ **Result**: Your question appears in the tree!

---

### Tutorial 2: Add Sub-Questions

**Scenario**: Break down your main question into specific areas.

**Steps**:
1. Hover over your main question
2. Click the **"+"** icon (Add Sub-Question)
3. Enter sub-question: "What types of exercise are most effective?"
4. Select type: **Sub**
5. Set status: **Investigating**
6. Click **"Create Question"**

**Repeat** for more sub-questions:
- "What is the optimal exercise duration?"
- "Does exercise timing matter?"
- "What are the mechanisms involved?"

‚úÖ **Result**: You now have a question hierarchy!

---

### Tutorial 3: Add a Hypothesis

**Scenario**: You have a hypothesis about aerobic exercise.

**Steps**:
1. Click on a question to expand it
2. Scroll to the **Hypotheses** section
3. Click **"Add Hypothesis"** button
4. Enter hypothesis: "Aerobic exercise improves memory through increased BDNF"
5. Select type: **Mechanistic**
6. Set status: **Testing**
7. Set confidence: **Medium**
8. Add description (optional)
9. Click **"Create Hypothesis"**

‚úÖ **Result**: Your hypothesis is linked to the question!

---

### Tutorial 4: Link Evidence to a Question

**Scenario**: You found a paper that supports your question.

**Steps**:
1. Click on a question to expand it
2. Click **"Link Evidence"** button
3. Search for the paper by title or author
4. Select the paper(s) you want to link
5. Choose evidence type: **Supports**
6. Rate relevance: **8/10**
7. Add key finding: "Study shows 30% improvement in memory tasks"
8. Click **"Link"**

‚úÖ **Result**: Evidence count updates, paper is linked!

---

### Tutorial 5: Link Evidence to a Hypothesis

**Scenario**: You found a paper that supports your hypothesis.

**Steps**:
1. Expand a question to see its hypotheses
2. Click **"Link Evidence"** on a hypothesis
3. Search and select paper(s)
4. Choose evidence type: **Supports**
5. Choose strength: **Strong**
6. Add key finding: "BDNF levels increased 40% after 12 weeks"
7. Click **"Link"**

‚úÖ **Result**: Hypothesis evidence counts update!

---

### Tutorial 6: View Linked Evidence

**Scenario**: Review all evidence linked to a hypothesis.

**Steps**:
1. Expand a question to see hypotheses
2. Click on the **evidence count badge** (e.g., "3 supporting")
3. Evidence list expands showing:
   - Paper titles (clickable)
   - Evidence type badges
   - Strength indicators
   - Key findings
4. Click paper title to view full paper
5. Click **X** to remove evidence if needed

‚úÖ **Result**: You can see all your evidence at a glance!

## üí° Best Practices

### 1. **Start with Main Questions**
- Begin with 3-5 broad main questions
- Break them down into sub-questions later
- Don't worry about perfection - you can edit anytime

### 2. **Use the Question Hierarchy**
- Main questions: Broad research areas
- Sub-questions: Specific aspects or methods
- Exploratory questions: Open-ended investigations

### 3. **Track Status Actively**
- **Exploring**: Just starting to investigate
- **Investigating**: Actively researching
- **Answered**: You have a clear answer
- **Parked**: On hold for now

### 4. **Link Evidence as You Read**
- Link papers immediately after reading
- Add key findings while they're fresh
- Use evidence types accurately:
  - **Supports**: Confirms your question/hypothesis
  - **Contradicts**: Challenges your question/hypothesis
  - **Neutral**: Relevant but doesn't support/contradict
  - **Context**: Background information
  - **Methodology**: Methods you might use

### 5. **Use Hypotheses Strategically**
- Link hypotheses to specific questions
- Track confidence levels honestly
- Update status as evidence accumulates
- Use strength indicators (weak/moderate/strong) to prioritize

### 6. **Review Regularly**
- Weekly: Review your question tree
- Update statuses and priorities
- Archive answered questions
- Add new questions as they emerge

## üêõ How to Report Bugs

If you encounter a bug, please report it with:

1. **What happened**: Brief description
2. **Steps to reproduce**: How to trigger the bug
3. **Expected behavior**: What should happen
4. **Screenshots**: If possible
5. **Browser & OS**: Chrome/Firefox/Safari, Mac/Windows

**Example**:
```
Bug: Can't delete hypothesis

Steps:
1. Created a hypothesis
2. Clicked delete button
3. Nothing happened

Expected: Hypothesis should be deleted
Actual: Nothing happens, no error message

Browser: Chrome 120, Mac OS 14.1
```
